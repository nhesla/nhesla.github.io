Dev Notes
What This Is

A 9×9 "Ultimate" Tic-Tac-Toe: a 3×3 outer grid of nine inner boards. Win three inner boards in a row to win the match. The inner cell you choose determines which outer board your opponent must play on next.

Originally written as a console C++ program. This is a faithful JS reimplementation of the same AI, ported to run in the browser with a click-based interface instead of coordinate input.

The Mark System

After every move, each inner board is re-evaluated and assigned a Mark enum value describing its strategic state. The AI reads these marks to decide which outer board to target.

X_mark Won by X
O_mark Won by O
Unwinnable Blocked for both players
Winnable Either player can win in one move
X_fork X has two ways to win
O_fork O has two ways to win
X_advantage X will fork regardless of O's play
O_advantage O will fork regardless of X's play
X_blockable X can win or fork in one move
O_blockable O can win or fork in one move
AI Decision Logic

The AI runs in two passes. First it picks which inner board to target on the outer 3×3, reading the Mark values above — prioritizing Winnable boards, then its own blockable state, then defending against the opponent's. If no priority target is found it falls back to a turns-to-win value score per position.

Inside the chosen board it runs the classic hierarchy: win if possible → block a win → fork → block a fork → center → opposite corner → empty corner → empty side. Each step is gated by a level-scaled probability roll — lower levels believably skip higher-priority plays rather than just playing randomly.

At level 9, winning and blocking are guaranteed. Forking and block-forking are near-certain but not absolute — the roll can still fail roughly 1-in-9 times. This is intentional and carried over from the original C++ design, where a comment in the source explicitly notes level 9 "can rarely make mistakes, theoretically making it possible to beat." A hidden level 15 existed in the original for a truly perfect AI.

Port Notes

The C++ version used coordinate pair console input (X=0, Y=1) for both the outer and inner board. Here that's replaced with click handlers — first click picks the sub-board, second picks the cell. The AI move fires after a short delay so the board doesn't update instantaneously.

system("CLS") and system("pause") calls are incompatible with a browser context; board state is re-rendered in place on each turn instead.
